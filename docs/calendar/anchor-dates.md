# Anchor Dates

An anchor date is the reference point from which dependent key dates are calculated. It's the date that lets DocJacket turn a reusable rule into a real transaction deadline.

## How it works

When you apply a checklist, each key date rule can be defined as an offset from an anchor date:

- "Inspection deadline: 10 days after anchor"
- "Appraisal deadline: 21 days after anchor"
- "Closing: 30 days after anchor"

The **anchor date** is usually the **contract date** — the day the purchase agreement was executed. But it could be any date that your key date rules reference.

## Setting the anchor date

When you apply a checklist, DocJacket uses the transaction dates already entered on the deal. It typically defaults anchor-based calculations to the contract date when that date is available.

You can change key dates later from the Timeline tab. When you do, DocJacket can recalculate dependent dates based on the original rules.

## Why this matters

The anchor date system means you don't have to manually calculate dozens of deadlines. Set the important transaction dates once, and dependent key dates flow from there. When a closing date gets pushed back or a contract date changes, recalculation can update the downstream timeline.

## Common anchor date scenarios

| Scenario | Typical anchor date |
|----------|-------------------|
| Standard residential purchase | Contract execution date |
| New construction | Permit approval date or contract date |
| Commercial lease | Lease commencement date |
| Short sale | Bank approval date |
