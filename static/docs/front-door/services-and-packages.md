# Services and Packages

Services and Packages power the public offering sections on your Front Door.

## Services

Go to **Front Door > Services** to manage the services catalog.

Each service can include:

- Name
- Short description
- Price
- Price qualifier such as starting at, flat, per deal, or per hour
- Timeline description
- Payment trigger
- Status: active, draft, or retired
- Public pricing visibility
- Scope notes
- Long description

Services can be reordered by dragging them in the services list. The public services section follows that ordering.

## Public pricing

Leave the price blank when the service should show as **Contact for quote**. Use **Hide price on public page** when the service should be public but the price should not be displayed.

## Service filters

Services can optionally be filtered by:

- Property type
- Transaction type
- Client type

Leaving a filter blank means the service applies broadly. Use filters when the same Front Door needs to support different kinds of prospects or intake paths.

## Checklist template link

A service can link to a checklist template. When an intake flow creates a transaction from that service, DocJacket can use the linked checklist template so the work plan is ready faster.

## Packages

Go to **Front Door > Packages** to bundle multiple services into one public offer.

Each package can include:

- Name
- Short description
- Bundle price
- Price qualifier
- Client type applicability
- Stage applicability
- Public visibility
- Status
- Included services
- Long description

Packages can show a savings amount when the individual service prices add up to more than the bundle price.

## When to use each

Use **Services** for individual offerings such as transaction coordination, listing prep, document review, or consultation.

Use **Packages** when you want to sell a bundle, such as a contract-to-close package that includes multiple services at one price.
