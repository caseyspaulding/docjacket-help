# Pending Review

Pending Review is the approval tray for AI-proposed work. Drafts wait here until a user approves, edits, or rejects them.

Drafts expire 72 hours after they are created.

## What appears here

Pending Review can include:

- Client update email drafts
- Document request drafts
- Agent follow-up drafts
- Proposed task bundles
- Proposed key date changes
- Transaction summaries

Drafts may come from connected AI clients, built-in DocJacket AI review, forwarded email workflows, or manual workflows.

## Filters

Use the source and type filters to narrow the tray.

Source filters include:

- AI clients
- QA Review
- Forwarded
- Manual

Type filters include:

- Emails
- Tasks
- Key dates
- Summaries

## Reviewing a draft

Open a draft to see:

- Source and artifact type
- Transaction, when linked
- Expiration time
- Draft content or proposed structured change
- Reason the draft was proposed

Email-shaped drafts and transaction summaries can be edited before approval. Structured drafts, such as task bundles and key date changes, are validated as proposed. Reject and ask for a new proposal if a structured draft needs different details.

## Approve, edit, or reject

Use:

- **Approve** to run the action
- **Edit** to adjust supported text drafts before approving
- **Save & approve** to edit and immediately approve
- **Reject** to archive the draft with an optional reason

For email drafts, approval sends the email right away using the organization's configured send mode.
