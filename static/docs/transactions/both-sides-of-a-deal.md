# Coordinating Both Sides of a Deal

When you're the TC for an agent who has **both sides** of a deal — representing buyer and seller, or a dual-agency listing — use **one transaction** with the **Side** set to **Dual** (some intake forms label it "Both Sides"). It's one physical closing, so one transaction keeps everything in one place.

## Why one transaction

A single transaction already holds both sides:

- **Both parties** — buyer(s) and seller(s)
- **Both agents** — listing agent and buyer's agent
- **Both commission sides** — listing-side and buyer-side amounts and rates
- **Both representatives**, one set of **key dates**, one **portal**, and one **document jacket**

Splitting one closing into two transactions means maintaining documents, contacts, dates, and portal links **twice**. DocJacket doesn't link two transactions or sync anything between them, so everything would be duplicated by hand.

## Setting it up

1. Create or open the transaction
2. Set **Side** to **Dual** / **Both Sides**
   - If "Dual" isn't in your Side options, add it under **Settings** (transaction field options), or pick "Both Sides (Dual)" on an intake form
3. Add the buyer-side and seller-side contacts and both agents
4. Enter the listing-side and buyer-side commission amounts

## One thing to do manually

Applying a checklist doesn't automatically stack two checklists for a Dual deal — it picks a single default. So **apply your buyer-side checklist and your seller-side checklist yourself** so both sets of tasks and deadlines land on the transaction. See [Checklists](../templates/timeline-templates.md).

## When two transactions make sense

Two separate transactions can be cleaner if your **reporting or exports assume one side per record** (for example, commission reports or side-based filters) and you're willing to maintain both files independently. Since there's no link between them, treat them as two separate deals.
