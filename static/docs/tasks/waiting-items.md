# Waiting Items

Waiting Items help you track things that are blocked on someone else. Use them for documents, confirmations, signatures, answers, or vendor updates that your team needs before moving forward.

Waiting Items are different from a transaction's **Waiting On** field. **Waiting On** is the single headline blocker on a transaction. **Waiting Items** are individual follow-up items that can be listed and resolved.

## Where to find them

Open **Waiting Items** from the sidebar to see all waiting items across transactions.

The list shows:

- Item title and description
- Related transaction
- Who you are waiting on
- Status
- Due date

Use the transaction link to open the related deal when you need more context.

## Marking an item received

When the item comes in, click the check action to mark it as received. The status updates so your team can see the blocker is cleared.

Statuses can include **Waiting**, **Received**, **Overdue**, or **Cancelled**. Only items still marked **Waiting** show the mark-received action.

## Tips

- Use waiting items for external blockers, not ordinary internal tasks.
- Add due dates when you need to follow up by a specific day.
- Keep descriptions short and specific so the list is easy to scan.
- Use the transaction's **Waiting On** field for the main blocker that should appear on the dashboard and transaction list.
